What MCP Servers Are (and Why They Matter)

MCP (Model Context Protocol) is an open standard created by Anthropic that lets AI assistants connect to external tools through a universal interface. An MCP server is a program that exposes a set of tools — query a database, search the web, create a GitHub issue, scan for vulnerabilities — that any compatible AI client can use.

Before MCP, every AI tool built its own integrations from scratch. Cursor had its GitHub integration. Claude Code had its own. They were incompatible. MCP changed that: one server works across all compatible clients. Build a PostgreSQL MCP server once, and it works in Claude Code, Cursor, Copilot, Windsurf, Zed, and 300+ other clients.

The numbers tell the story: the MCP SDK hit 97 million monthly downloads in March 2026. Over 10,000 public servers exist. The official reference server monorepo has 86,000+ GitHub stars. MCP is to AI tools what USB was to peripherals — the universal connector that made the ecosystem explode.

The problem: with 10,000+ servers, finding the good ones is hard. Most are abandoned experiments or thin wrappers. This guide is the curated list — 40+ servers that are actively maintained, well-documented, and genuinely useful, organized by what you actually need them for.

How to Install MCP Servers

MCP servers run locally on your machine. You configure them in your AI tool’s settings, and the tool starts them automatically when needed.

Claude Code

Add servers to .mcp.json in your project root (per-project) or ~/.claude/mcp.json (global). Full docs.

// .mcp.json (project root)
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "./src"]
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": { "GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_..." }
    },
    "postgres": {
      "command": "npx",
      "args": ["-y", "@crystaldba/postgres-mcp"],
      "env": { "DATABASE_URI": "postgresql://localhost:5432/mydb" }
    }
  }
}

Claude Desktop

Add to claude_desktop_config.json. Same format as above. Location: ~/Library/Application Support/Claude/claude_desktop_config.json on macOS.

Cursor

Go to Settings → MCP Servers → Add Server. Cursor MCP docs.

VS Code (Copilot)

Add to .vscode/mcp.json or user settings. VS Code MCP docs.

MCP Server Safety Guide

MCP servers run locally with your user permissions. A poorly written or malicious server can read your files, execute commands, or exfiltrate data. Here’s how to stay safe:

  • Prefer official servers. Servers published by the tool vendor (GitHub’s own server, Sentry’s own server) are the safest bet. They have the most eyes on the code and the strongest incentive to maintain trust.
  • Audit the tool list. A server that exposes write_file or execute_command has more risk surface than one that only reads data. Know what you’re granting access to before enabling a server.
  • Use read-only modes. Some servers (like Postgres MCP Pro) support read-only modes. Use them when your agents only need to query, not modify.
  • Pin versions. Don’t use npx -y server@latest in production. Pin to a specific version so updates don’t introduce unexpected behavior.
  • Check stars and maintenance. A server with 10 stars and no commits in 6 months is riskier than one with 1,000 stars and weekly releases. Every server in this guide is actively maintained.
  • Minimize secrets. Use the narrowest API permissions possible. A GitHub token for MCP should be scoped to specific repos, not your entire organization.

Security research from BlueRock found SSRF vulnerabilities in 36.7% of surveyed MCP servers. The servers in this guide were selected in part for their security posture, but always review a server’s permissions before trusting it with sensitive data.

MCP Servers + amux

amux is an agent control plane that manages fleets of AI coding agents. MCP servers extend what those agents can do. Here’s what makes amux’s MCP integration different:

  • Per-session MCP configs. Different agents get different tool sets. Your database agent gets the Postgres server. Your frontend agent gets the Playwright MCP. Your security agent gets Semgrep. Each session’s MCP config is independent.
  • Centralized config. One mcp.json in the project root, shared by all sessions that need the same tools. No duplicating config across 10 agents.
  • Built-in REST API as an MCP alternative. amux’s own REST API (board, sessions, notes, CRM) is available to all agents without MCP. Agents can coordinate via the board, message each other, and share state — capabilities that would otherwise require building a custom MCP server.

Frequently Asked Questions

What is an MCP server?

An MCP (Model Context Protocol) server is a program that exposes tools, resources, and prompts to AI assistants like Claude Code, Cursor, GitHub Copilot, and Windsurf via a standardized protocol. Instead of each AI tool building its own integrations, MCP provides a universal interface — one server works across all compatible clients. Think of it as a USB-C port for AI tools: any MCP server plugs into any MCP client.

How do I install an MCP server?

Most MCP servers install via npx (Node.js) or uvx (Python). Add the server to your AI tool’s MCP config file: claude_desktop_config.json for Claude Desktop, .mcp.json for Claude Code, or the MCP settings panel in Cursor/Windsurf. Each server’s GitHub README has the exact config snippet. See the installation section above for examples.

Which MCP clients support MCP servers?

Over 300 MCP clients exist as of May 2026. The major ones: Claude Desktop, Claude Code, Cursor, GitHub Copilot (VS Code), Windsurf, Zed, Cody by Sourcegraph, Replit, and Amazon Q Developer. See the compatibility matrix for feature support details.

How many MCP servers exist?

Over 10,000 public MCP servers exist as of May 2026, up from roughly 1,200 a year ago. The MCP SDK hit 97 million monthly downloads in March 2026. The official reference server monorepo has 86,000+ GitHub stars.

Are MCP servers safe to use?

MCP servers run locally with your user permissions, so trust matters. Stick to official servers published by the tool vendor or well-starred community servers with active maintenance. Audit the tool list before enabling — a server with write_file or execute_command has more risk than read-only servers. Security research from BlueRock found SSRF vulnerabilities in 36.7% of surveyed servers. See our safety guide for best practices.

Can I use multiple MCP servers at once?

Yes. Most clients support multiple concurrent servers. A typical setup: filesystem + GitHub + database + search — all active simultaneously. The AI assistant sees tools from all connected servers and can use them together. With amux, you can configure different MCP server sets per session for role-based access.

What is the difference between MCP and A2A?

MCP connects agents to tools and data sources — it answers “what can this agent do?” A2A (Agent-to-Agent Protocol by Google) connects agents to each other — it answers “how do agents collaborate?” They’re complementary. An agent uses MCP to access a database, and A2A to delegate a subtask to another agent. See our A2A + MCP guide.
